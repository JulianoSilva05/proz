Desafio do Mundo Real: Sistema de Agendamento de Salas

Empresa: InnovateTech Solutions

Sua Posição: Desenvolvedor(a) Júnior

Sua Missão: Criar um sistema interno para agendamento de salas de reunião.

1. Cenário

A InnovateTech tem 3 salas de reunião: "Alfa", "Beta" e "Gama". Atualmente, o agendamento é feito em uma planilha compartilhada, o que está causando muitos conflitos de horários e confusão. Sua missão é criar uma página web simples onde qualquer funcionário possa ver os agendamentos existentes e criar um novo agendamento, evitando conflitos de horário.

2. Tecnologias Envolvidas

  • Backend: PHP
  • Banco de Dados: MySQL
  • Frontend: HTML e CSS

3. Requisitos Funcionais

  1. Página Principal (index.php):
    • Deve exibir a lista de salas de reunião disponíveis.
    • Para cada sala, deve haver um link ou botão para "Ver Agendamentos".
    • Deve haver um formulário para criar um novo agendamento.
  2. Página de Detalhes da Sala (sala.php):
    • Ao clicar em "Ver Agendamentos", o usuário deve ser levado para uma página que mostra todos os agendamentos para aquela sala específica.
    • A página deve exibir o nome do responsável, a data, a hora de início e a hora de fim de cada agendamento.
  3. Lógica de Agendamento (agendar.php):
    • O formulário deve enviar os dados (sala, nome do responsável, data, hora de início, hora de fim).
    • Regra de Negócio Principal: O sistema não pode permitir que um novo agendamento seja criado se o horário solicitado se sobrepuser a um agendamento já existente para a mesma sala.
    • O sistema deve fornecer feedback claro ao usuário (sucesso ou erro).

4. Estrutura do Banco de Dados (MySQL)

Sugerimos um banco de dados chamado innovatetech_agendamentos com as seguintes tabelas:

Tabela `salas`

CREATE TABLE salas (
    id INT AUTO_INCREMENT PRIMARY KEY,
    nome VARCHAR(100) NOT NULL
);

-- Inserindo os dados iniciais
INSERT INTO salas (nome) VALUES ('Sala Alfa'), ('Sala Beta'), ('Sala Gama');

Tabela `agendamentos`

CREATE TABLE agendamentos (
    id INT AUTO_INCREMENT PRIMARY KEY,
    id_sala INT NOT NULL,
    responsavel VARCHAR(255) NOT NULL,
    data_agendamento DATE NOT NULL,
    hora_inicio TIME NOT NULL,
    hora_fim TIME NOT NULL,
    data_criacao TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (id_sala) REFERENCES salas(id)
);

5. Passos Sugeridos para a Implementação

  1. Configuração do Ambiente: Crie o banco de dados e as tabelas. Prepare seu arquivo de conexão PHP (conexao.php) usando PDO ou MySQLi.
  2. Listar Salas e Criar Formulário (index.php): Use PHP para buscar e exibir as salas do banco de dados. Monte o formulário de agendamento.
  3. Processar o Agendamento (agendar.php): Receba os dados do formulário, valide as informações e implemente a lógica de verificação de conflito de horários antes de inserir no banco.
  4. Exibir Agendamentos (sala.php): Crie a página que recebe um ID de sala pela URL e exibe todos os seus agendamentos em uma tabela.
  5. Estilização (CSS): Crie um arquivo estilo.css para dar uma aparência limpa e profissional ao sistema.

6. Desafios Extras (Para ir além)

  • Editar e Excluir: Adicione funcionalidades para editar ou excluir agendamentos existentes.
  • Validação no Frontend: Use JavaScript para validar o formulário antes do envio (ex: a hora de fim não pode ser anterior à de início).
  • Segurança: Implemente o uso de Prepared Statements em todas as suas consultas SQL para prevenir Injeção de SQL.

Boa sorte com o desafio!
